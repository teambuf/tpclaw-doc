# 示例：分析项目代码

在聊天框中直接告诉智能体项目路径，它会自动读取目录结构、关键文件，生成项目分析报告。

## 使用方法

输入项目路径和分析需求，例如：

> D:\github\aa\xxx 帮我分析这个项目

智能体会自动完成以下操作：

1. **读取目录结构** — 调用 `read` 工具扫描项目文件列表
2. **读取关键文件** — 读取 README.md、配置文件等了解项目信息
3. **生成分析报告** — 汇总项目概览、技术栈、目录结构、功能模块等

## 智能体做了什么

| 步骤 | 工具 | 说明 |
|------|------|------|
| 扫描目录 | `read` | 读取项目根目录的文件和子目录列表 |
| 读取 README | `read` | 获取项目的自述文件内容 |
| 综合分析 | LLM | 根据目录结构和文件内容生成项目分析 |

## 其他可用的分析指令

你还可以提出更具体的分析需求：

| 指令 | 说明 |
|------|------|
| `分析这个项目的架构设计` | 重点关注架构和模块划分 |
| `帮我看看这个项目用了哪些技术栈` | 聚焦技术栈和依赖 |
| `这个项目的入口文件在哪，启动流程是什么` | 分析启动流程 |
| `帮我梳理这个项目的 API 接口` | 分析接口定义 |
| `这个项目有什么潜在问题或改进建议` | 代码审查和建议 |

> 以上示例在网页聊天和飞书/企业微信中都可以使用，用法完全一样。
