# 定时任务

通过自然语言对话即可创建和管理定时任务，智能体会自动调用 cron 工具完成操作。

## 概述

定时任务可以让智能体在指定时间自动执行操作，例如：
- 定期检测系统状态
- 定时发送通知到飞书/企业微信
- 周期性执行数据检查

> 无需记忆命令和 Cron 表达式，直接用自然语言描述你的需求即可。

---

## 使用示例

### 创建定时任务

直接告诉智能体你要做什么，例如：

> 帮我设置一个定时任务，每5分钟检测电脑状态，然后通过飞书渠道发给我

![设置定时任务](/img/cron/0.设置定时任务.png)

智能体会自动完成：
1. 解析你的需求（频率、内容、通知方式）
2. 创建定时任务并启用
3. 返回任务 ID、频率、状态等信息

### 查看定时任务

> 我当前有哪些定时任务

![查看定时任务](/img/cron/1.查看有那些定时任务.png)

智能体会列出当前所有定时任务的名称、ID、频率和状态。

### 修改定时任务

> 帮我把这个定时任务修改成2分钟一次

![修改定时任务](/img/cron/2.修改定时任务.png)

智能体会根据你的描述更新任务的执行频率。

### 飞书收到推送

定时任务触发后，智能体会执行检测并将结果推送到飞书：

![飞书收到推送](/img/cron/3.飞书收到推送信息.png)

## 常见场景

| 场景 | 示例指令 |
|------|----------|
| 每日提醒 | "每天早上9点提醒我开早会" |
| 工作日打卡 | "工作日早上9点提醒我打卡" |
| 系统监控 | "每10分钟检查服务器状态，有异常发飞书通知我" |
| 定时报告 | "每天下午6点生成今日工作总结发到飞书" |
| 周期检查 | "每小时检查一次待办事项" |

> 以上指令在网页聊天和飞书/企业微信中都可以使用，用法完全一样。

## Cron 表达式参考

如果需要精确控制执行时间，可以使用 Cron 表达式。格式：`秒 分 时 日 月 周`

### 常用表达式

| 表达式 | 描述 |
|--------|------|
| `0 */5 * * * *` | 每 5 分钟 |
| `0 0 8 * * *` | 每天早上 8 点 |
| `0 0 9 * * 1-5` | 工作日早上 9 点 |
| `0 0 */2 * * *` | 每 2 小时 |
| `0 30 12 * * *` | 每天中午 12:30 |
| `0 0 18 * * 1,3,5` | 周一、三、五下午 6 点 |

### 特殊字符

| 字符 | 含义 | 示例 |
|------|------|------|
| `*` | 任意值 | `* * * * * *` = 每秒 |
| `/` | 间隔 | `*/5` = 每 5 单位 |
| `-` | 范围 | `1-5` = 1 到 5 |
| `,` | 列举 | `1,3,5` = 1、3、5 |
