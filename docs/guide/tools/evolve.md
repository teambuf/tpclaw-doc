# evolve - 技能进化

将有价值的执行过程沉淀为可复用技能，实现智能体能力的持续积累。

## 触发方式

- **用户主动触发**：说"保存为技能"、"沉淀一下"、"以后都这么干"
- **智能体主动建议**：判断当前执行有复用价值时

## 是否值得沉淀

评估标准（满足至少 2 项才值得沉淀）：

| 标准 | 说明 |
|------|------|
| **可复用** | 同样的流程未来还会用到，不是一次性操作 |
| **多步骤** | 包含 3 个以上步骤的复杂操作 |
| **有门槛** | 包含容易出错的细节或隐藏依赖 |
| **通用性** | 不依赖当前特定环境，可迁移到类似场景 |

### 不适合沉淀的情况

- 单条命令或单步操作
- 完全依赖当前上下文的操作
- 简单问答或知识查询
- 已有现成工具覆盖的操作

## 使用流程

### 1. 触发沉淀

在完成一个复杂流程后，告诉智能体：

```
保存为技能
```

或者智能体会主动建议：

```
这个流程比较复杂，建议沉淀为技能以便复用。要继续吗？
```

### 2. 生成草稿

智能体会自动提取执行过程，生成技能草稿：

```
技能草稿已生成！
- 技能名称：{skillName}
- 描述：{description}
点击查看详情并审核：[查看技能草稿](/skills/review/{agentId}/{skillName})
```

### 3. 审核草稿

有两种审核方式：

**方式一：在对话中审核**

直接回复以下指令：
- **查看** - 查看草稿全文
- **同意** - 审核通过，保存为正式技能
- **拒绝** - 拒绝并删除草稿

也可以直接回复修改意见，智能体会根据反馈调整草稿内容。

**方式二：在技能管理页面审核**

在智能体的技能管理页面，草稿状态的技能会显示审核入口，可以查看详情并进行审核操作。

### 4. 技能生效

审核通过后，技能自动保存到智能体的技能目录，后续对话中可直接使用。

## 技能状态

| 文件 | 状态 | 说明 |
|------|------|------|
| 只有 `SKILL.md` | active | 正式生效 |
| 只有 `DRAFT.md` | draft | 待审核 |
| 两者都有 | draft | 优先显示草稿 |

## 技能存储位置

- **主智能体**：`skills/{skillName}/`
- **其他智能体**：`../workspace-{agentId}/skills/{skillName}/`

---

## 最佳实践

### 适合沉淀的场景

1. **多工具协作流程**
   ```
   分析项目代码 → 找到问题 → 生成修复方案 → 应用修改
   ```

2. **复杂配置流程**
   ```
   读取配置 → 验证格式 → 修改参数 → 重启服务 → 验证结果
   ```

3. **重复性调试流程**
   ```
   查看日志 → 定位错误 → 分析原因 → 尝试修复 → 验证
   ```

### 技能命名建议

- 使用小写字母、数字和中划线
- 简洁明了，体现核心功能
- 示例：`debug-flow`、`deploy-check`、`code-review`

---

## 注意事项

1. **审核是必须的**：所有技能草稿都需要用户审核后才能生效
2. **可以修改**：审核前可以要求智能体修改草稿内容
3. **不要滥用**：只沉淀真正有价值的流程，避免技能膨胀
4. **保持更新**：随着流程变化，及时更新已有技能
